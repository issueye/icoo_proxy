## 1. 规则命中调试器

### 目标
用户在“对话规则”页面输入样例请求后，可以看到最终路由结果和命中原因。

### 后端任务
- [ ] 新增规则调试请求结构：`DebugRouteRequest`
  - model
  - system
  - messages
  - stream
- [ ] 新增规则调试响应结构：`DebugRouteResult`
  - matched
  - matchedRule
  - matchedReason
  - providerId
  - providerName
  - targetModel
  - fallbackReason
- [ ] 在服务层新增 Wails 方法：`DebugRoute(...)`
- [ ] 复用现有 provider 路由决策逻辑，避免复制一套规则判断
- [ ] 为规则匹配过程补充可解释信息
- [ ] 未命中规则时返回默认供应商 / 默认模型决策
- [ ] 增加单元测试覆盖：
  - 模型名匹配
  - system 包含匹配
  - user message 包含匹配
  - 无命中规则
  - 多规则优先级

### 前端任务
- [ ] 在 `DialogRulesView.vue` 增加规则调试面板
- [ ] 增加测试模型输入框
- [ ] 增加 system 输入框
- [ ] 增加 user message 输入框
- [ ] 增加“模拟路由”按钮
- [ ] 增加调试结果展示区
- [ ] 展示命中规则、目标供应商、目标模型、命中原因
- [ ] 未命中时展示默认路由说明
- [ ] 增加 loading、错误、空状态

### 验收
- [ ] 用户能在页面上输入样例并得到稳定路由结果
- [ ] 命中原因能解释到具体匹配字段
- [ ] 不影响现有规则保存功能

## 完成定义

一个任务只有同时满足以下条件，才视为完成：

- [ ] 功能可在界面或接口稳定使用
- [ ] 关键异常路径有明确错误提示
- [ ] 不破坏现有配置和旧数据
- [ ] 至少覆盖核心单元测试或手动验收用例
- [ ] 文档状态已同步更新
