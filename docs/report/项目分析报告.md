# icoo_proxy 项目分析报告

> 生成时间：2026-07-14（本地时间 CST，UTC+08:00）
> 分析范围：仓库根目录、icoo_llm_bridge、icoo_desktop、docs、images、icoo_proxy

## 1. 项目定位
`icoo_proxy` 是一个**本地优先的 LLM API 桥接与桌面管理端**。它在下游暴露 OpenAI Chat Completions、OpenAI Responses、Anthropic Messages 兼容接口，并按供应商、模型、路由规则将请求转发到不同上游；必要时做协议转换。后端核心在 `icoo_llm_bridge`，桌面控制台在 `icoo_desktop`，最终交付目录为 `icoo_proxy`。

## 4. 入口与运行
- 网关入口：`icoo_llm_bridge/cmd/bridge/main.go`
- 桌面入口：`icoo_desktop/main.go`
- 默认监听：`127.0.0.1:18181`
- 健康检查：`/healthz`、`/readyz`
- 本地默认允许回环免鉴权访问管理接口（文档明确不建议暴露到非本机网络）。

## 5. 接口与路由
### 代理入口
- `POST /v1/chat/completions`
- `POST /v1/responses`
- `POST /v1/messages`
- 动态入口通过已启用 `IngressEndpoint` 路由

### 管理入口
- 运行时状态：`GET /api/v1/runtime/state`
- 供应商/模型：`/api/v1/providers`、`/api/v1/providers/:id/models`、`/api/v1/model-catalog`
- 端点/规则：`/api/v1/ingress-endpoints`、`/api/v1/routing-rules`
- 授权 Key：`/api/v1/api-keys`
- 流量：`/api/v1/traffic`
- UI 偏好：`/api/v1/ui-prefs`

## 6. 模块边界
### 后端
- `cmd/bridge`：启动参数与生命周期
- `internal/app`：容器组装
- `internal/config`：配置加载
- `internal/router`：路由注册
- `internal/controller`：HTTP 适配
- `internal/service`：业务编排、代理链路、路由、供应商、流量
- `internal/repository`：GORM 仓储
- `internal/model`：领域对象与数据库实体
- `internal/utils/ai_llm_proxy`：纯协议转换工具包

### 桌面端
- `main.go` + `app.go`：Wails 宿主、tray、bridge 进程启停
- `frontend/src/stores`：Pinia 状态层
- `frontend/src/views`：业务页面
- `frontend/src/lib/apiClient.js`：后端通信入口

## 7. 数据存储与配置
- 主库：`.data/icoo_llm_bridge.db`
- 流量库：`.data/icoo_llm_bridge_traffic.db`
- 自动迁移表：Provider、ProviderModel、ModelCatalogItem、IngressEndpoint、RoutingRule、APIKey、UIPreference、TrafficRecord
- SQLite 连接策略：单连接、WAL、busy_timeout、synchronous=NORMAL
- 配置样例：`icoo_llm_bridge/configs/config.example.toml`
- 链路日志：`.data/bridge-chain.log`
- 可选归档：`.data/down_request`、`.data/up_request`

## 8. 构建与运行
### 网关
```powershell
cd icoo_llm_bridge
.\build.ps1
.\build\bridge.exe
```

### 桌面端
```powershell
cd icoo_desktop
.\build.ps1 -BridgePath ..\icoo_llm_bridge\build\bridge.exe
.\build\bin\icoo_desktop.exe
```

### 一键打包
```powershell
.\build-all.ps1
```

输出：
- `icoo_proxy\bridge.exe`
- `icoo_proxy\icoo_desktop.exe`

## 10. 风险与改进建议
### 高风险
- 本地回环默认免鉴权，若 host 暴露后缺少充分防护，存在未授权访问风险。
- 供应商密钥在部分流程中可能被暴露或更新语义不正确。
- 跨协议流式转换仍存在缓冲，部分路径非低延迟。
- 取消请求状态码分类不一致，可能影响流量诊断。

### 中风险
- SQLite 单连接写队列在峰值下可能成为吞吐瓶颈。
- 供应商健康检查与模型发现前后端协议曾漂移，说明接口契约稳定性不足。

### 改进建议
- 优先执行 2026-07-10 计划任务。
- 增加 origin / 来源校验与更细粒度的鉴权策略。
- 统一取消状态码为 `499` 并在流量面板独立展示。
- 统一桌面端与网关版本号。
- 补充 OpenAPI / CHANGELOG / LICENSE。
- 增加持续构建与发布验证流程。
