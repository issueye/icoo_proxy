# icoo_proxy TODO 计划文档

## 目标

围绕当前项目的三条主线推进：

1. 提升路由结果可解释性
2. 提升日志诊断与可观测性
3. 提升供应商管理自动化与稳定性

---

## P0：优先落地

## 1. 规则命中调试器

### 产品目标
让用户能直接验证一条请求最终会命中哪条规则、路由到哪个供应商、使用哪个目标模型。

### TODO
- [ ] 在“对话规则”页面新增“规则调试”区域
- [ ] 支持输入测试模型名
- [ ] 支持输入 system message
- [ ] 支持输入 user message
- [ ] 支持点击“模拟路由”
- [ ] 返回命中规则名称
- [ ] 返回命中原因说明
- [ ] 返回目标供应商
- [ ] 返回最终目标模型
- [ ] 未命中时返回默认路由结果
- [ ] 对冲突规则给出基础提示

### 验收标准
- 能输入一组样例请求并稳定返回路由结果
- 用户能明确看到“为什么命中这条规则”
